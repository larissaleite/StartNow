New Repository


1. Create repository in https://github.com/<username>

2. Create folder in the local machine, preferrably with the same name as the repository

3. Reach the folder using the command line and then type the following:

git --global user.name "<username>"

git --global user.email "<email>"

This command is useful so that you don't have to type in your username and password everytime you push new changes.

git init

git remote add origin https://github.com/<username>/<repository name>

This command initiates the local repository in the folder just created and links it to the remote repository created in the first step.

touch README.md

git add README.md

The first command simply creates an empty README file, whereas the second one adds the file to be sent to the remote repository.

git commit -m "<Comment>"

git push -u master

Finally, a message is added to the commit, to make it easier to keep track of what happened in each change to the repository, and all the changes are sent (pushed) to the repository.

Existing Repository


1. When there is already a repository up and running either created by you or someone else (you must be added as a collaborator though), firstly create a folder with the same name as the repository in the local machine.

2. Then, just clone the repository:

git clone https://github.com/<username>/<repository name>.git

Using GitHub Pages


1. Create new branch in https://github.com/<username>/<repository name>

2. In the command line type:

git fetch

This command is necessary so that the local repository becomes aware of the new branch created.

git checkout gh-pages

git merge master

The first command changes the branch from master to gh-pages, and the second one merges the content updating gh-pages with all the files in master.
